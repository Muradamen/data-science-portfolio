# 🐍 Python Foundations for Data Processing

## 📌 Overview

This project represents the engineering foundation for building scalable data systems. It transitions from basic Python scripting to **modular, object-oriented design**, enabling the development of maintainable and production-ready data solutions.

The work demonstrates how core programming concepts evolve into real-world system design using Python.



## 📂 Project Modules

### 1️⃣ Logic & Data Structures (`Python_Variables,_Data_Types_and_Control_Structures.ipynb`)

Focuses on the building blocks of data processing and algorithmic thinking.

**Key Concepts:**

* Variables and data types
* Data structures (Lists, Dictionaries, Sets, Tuples)
* Conditional statements and control flow
* Loops and iteration
* Functions and modular programming
* Python modules and packages



### 2️⃣ Object-Oriented Design (`Object_Oriented_Programming.ipynb`)

Focuses on designing scalable systems using Object-Oriented Programming (OOP).

**Key Concepts:**

* Classes and objects
* Encapsulation
* Inheritance
* Polymorphism
* Abstraction
* Code modularity and reuse



## 🧱 Core OOP Principles (A PIE)

* **Abstraction:** Simplifying complex systems by exposing only essential features
* **Encapsulation:** Protecting data and controlling access through class design
* **Inheritance:** Reusing code through hierarchical relationships
* **Polymorphism:** Designing flexible interfaces for multiple implementations


## 💡 Applied Problem: E-Commerce Management System

### 📖 Context

To bridge theory with real-world application, an **Object-Oriented E-Commerce System** was designed to simulate business operations in an online retail environment.



## 🎯 Problem Statement

Design a modular Python system that models products, customers, orders, and payments while demonstrating core OOP principles.


## ⚙️ System Design

### 🛍️ Product

* Attributes: name, price, size, color, quantity
* Methods: purchase, update stock, delivery

### 👤 Customer

* Stores and manages customer data
* Interacts with orders

### 📦 Order

* Handles product selection
* Calculates total cost
* Manages checkout process

### 💳 Payment

* Multiple payment methods (PayPal, Bank Transfer)
* Demonstrates polymorphism via interchangeable payment behaviors



## 🔄 Workflow Simulation

1. Customer selects products
2. Order is created
3. Total is calculated
4. Payment method is selected
5. Order is completed



## 🛠️ Technical Stack

* Python 3.x
* Jupyter Notebook
* Modular programming & OOP design


## 🚀 Key Engineering Takeaways

* Transitioned from scripting to **modular software design**
* Built reusable components using **OOP principles**
* Structured code for **scalability and maintainability**
* Applied programming concepts to a **real-world system**
